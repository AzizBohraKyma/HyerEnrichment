## Bug found & fixed: "Event loop is closed" in RQ worker

E2E job failed with `RuntimeError: Event loop is closed` inside `check_suppression`.
Root cause: `run_enrichment_job` uses `asyncio.run` per job (fresh loop each time),
but the async Redis client (`app/storage/redis_client.py`) and the SQLAlchemy engine
pool are module-globals — connections created in job N's loop leak into job N+1's
loop and blow up. Fix: `_run_enrichment_job` now does `await close_redis()` and
`await engine.dispose()` in a `finally`. Same reason `_init_db_once` in `rq_worker.py`
disposes the engine after `create_all`.

## Free-now / paid-swappable enrichers (all phases)

Implemented the full "free now, paid-swappable later, zero orchestrator refactor" plan.

- **Phase 0 provider layer** `app/providers/`: `proxy.py` (ProxyProvider), `browser.py`
  (BrowserProvider, lazy Playwright), `llm.py` (LLMDecision + heuristic/ollama/litellm +
  Langfuse trace no-op), `email_verify.py` (EmailVerifier basic->smtp chain), `sidecar.py`
  (SidecarClient, unreachable->None), `process.py` (`run_command`, never raises).
- **config.py**: 5 mode flags + per-tier optional vars; all default to free mode. `extra="ignore"`
  meant `.env` could carry them before fields existed, but I added the `Field(...)` entries too.
- **Enrichers**: base `Enricher` now has a concrete `run` template that wraps a new abstract
  `_fetch` and tags sources + swallows failures. All 11 enrichers rewritten to real
  subprocess/library/sidecar calls that degrade to empty fragments. Tier 1 gated by `ENABLE_TIER1`.
- **Design rule kept**: orchestrator, Enricher public protocol, and Dossier shape unchanged.
  free->paid is purely an env flip.
- **Compose**: real images; free sidecars (social-analyzer, google-maps-scraper) default-on,
  paid/heavy (reacher, litellm, ollama, scrapoxy, langfuse, changedetection) behind `profiles:`.
  Langfuse put behind a profile (needs its own datastore) — noted as a deliberate deviation.
- **New route** `POST /api/signals/changedetection` (unauth by design, optional shared-secret header).
- **Tests**: added `tests/test_enrichers.py` (mocked seams). Updated `test_pipeline_shape.py` with an
  autouse `_offline_enrichers` fixture that stubs every `_fetch` so the full pipeline stays offline
  and deterministic. `pytest tests` -> 18 passed; `ruff check` clean.

### Lesson

- The existing pipeline shape tests asserted `dossier.handles` truthy with no mocking (old enrichers
  returned mock payloads). Real enrichers degrade to empty offline, so I had to add the offline stub
  fixture to keep those assertions meaningful AND honor "no live external calls in CI".
- Chose subprocess over library API for sherlock/maigret (their programmatic APIs are heavy/unstable);
  left `# NOTE:` comments per RULE's "when to break a rule".
