# 2026-07-10

## Docker Tier 1 diagnosis (WSL2 + Docker Engine + Multilogin on Windows)

Ran the networking checklist against live stack.

**Works from Docker worker:** Multilogin launcher `https://launcher.mlx.yt:45001` (HTTP 404), all Tier1/R2/Redis/Postgres env vars present via `docker-compose.tier1.yml`, Redis/SQL URLs point at compose services.

**Breaks:** After `start_profile` returns a Selenium port (e.g. 55355 / 50947), TCP from container/WSL to `172.26.128.1:<port>` **times out**. Same port on container `127.0.0.1` = connection refused. RQ job then hits 180s `JobTimeoutException` during `connect_selenium`.

**Root cause:** Launcher API is reachable on host IP; Multilogin Selenium automation port is not reachable from Docker/WSL (bound to Windows loopback and/or firewall). Local Windows Python works because it uses `127.0.0.1` on the Windows host itself.

**Not the cause this time:** missing env, R2 creds, Redis URL, wrong compose secrets, Python version, missing Chromium in image (remote Selenium), volume mounts.
